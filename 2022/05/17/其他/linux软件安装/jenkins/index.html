jenkins

一、介绍

1.1 持续集成流程说明

  1. 首先开发人员代码提交，提交到 Git 仓库
  2. 然后 Jenkins 作为持续集成工具，使用 Git 工具到 Git 仓库到集成服务器，配合 JDK,Maven 等工具完成代码编译，代码测试与审查，打包等工作。每一个流程出差，都会重新执行一次整个流程
  3. 最后，jenkins 把生成的 jar 或者 war 包分发到测试服务器或者生产服务器。测试人员或者用户就可以访问这个应用

1.2 安装

1.2.1 环境准备

  • JDK 环境

java -version

  • Maven 环境

mvn -version
